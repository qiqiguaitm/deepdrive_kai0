# 双臂 VLA 数据采集 UI —— 设计计划

> 目标：为 kai0 (χ₀) 项目提供一套可视化数据采集界面，覆盖 Agilex 双 Piper 双臂 + 三路 RealSense 相机的遥操作录制流水线，产出严格对齐 LeRobot v2.1 的数据集。

---

## 〇、角色模式（采集员 / 管理员）

系统提供两种使用模式，顶栏一键切换，**均无需密码**：

### 1. 采集员模式（Collector，默认）
- 入口选择操作员姓名即可进入。
- 只暴露与「采集」强相关的功能：相机预览、双臂状态、录制控制、**任务/Prompt 下拉选择**、episode 元信息、当次 episode 回放、历史 episode 浏览 / 回放、已录数据统计、磁盘剩余提示。
- **不暴露**：rerun 可视化、任务/Prompt 模板编辑、历史数据删除/重命名、标定/配置修改、模型权重路径切换、质检阈值调整、后端服务重启。
- UI 上方显示「采集员模式」徽标 + 当前操作员。

### 2. 管理员模式（Admin）
- 顶栏「切换到管理员」直接进入，无密码（本机内网部署，信任环境）。
- **额外解锁**：
  - **任务 / Prompt 模板管理**：新增 / 编辑 / 禁用任务项与 prompt 条目（供采集员下拉使用）。
  - **Rerun 可视化**：对任意已采 episode 或实时流启动 Rerun viewer（三路视频 + 14 维关节曲线 + 动作指令 + 时间戳对齐），复用 `docs/deployment/inference_visualization.md` / `inference_visualization_mesh.md` 管线，按需加载 mesh 版本。
  - 历史 episode 打标 / 删除 / 重命名 / 批量导出。
  - 质检阈值、相机/CAN、存盘路径等配置修改。
  - 后端诊断：ROS2 话题健康、录制 worker 日志、服务重启。
- UI 上方显示「管理员模式」徽标（红色），破坏性操作二次确认。

### 3. 权限矩阵（摘要）

| 能力 | 采集员 | 管理员 |
|---|---|---|
| 实时相机/关节预览 | ✅ | ✅ |
| 开始 / 保存 / 丢弃录制 | ✅ | ✅ |
| 任务 / Prompt 下拉选择 | ✅ | ✅ |
| **任务 / Prompt 模板编辑** | ❌ | ✅ |
| 回放刚采 episode | ✅ | ✅ |
| 浏览 / 回放任意历史 episode | ✅ | ✅ |
| 已录数据统计看板 | ✅（只读） | ✅ |
| Rerun 可视化（实时 & 历史） | ❌ | ✅ |
| 历史 episode 删除 / 重命名 / 导出 | ❌ | ✅ |
| 配置 / 阈值 / 路径修改 | ❌ | ✅ |
| 后端服务控制 | ❌ | ✅ |

## 二、功能分区

### 1. 顶部状态栏（StatusBar，全局常驻）
独立顶栏，与业务面板解耦，单向接收后端 `/ws/status` 推送。字段：
- **角色与操作员**：当前模式徽标 + 操作员姓名。
- **连接健康**：ROS2 节点在线、双臂 CAN 链路（CAN-L / CAN-R）、三路相机（3/3）、遥操作主从握手。
- **录制状态机**：`IDLE` / `REC ●` / `SAVING` / `ERROR`；录制中显示已录时长。
- **当前任务与 Prompt**：反映下拉选择结果。
- **Episode 编号**：下一条 episode 的递增号（由后端扫描真实文件决定，避免重复）。
- **磁盘与写入速率**：剩余空间、瞬时写入 MB/s。
- **系统级告警**：任何子系统异常（丢帧、CAN 心跳超时、磁盘 <10 GB）在状态栏高亮。
- 断连时整栏变灰 + 红点闪烁；点击任一指示灯跳转到对应诊断面板（管理员模式）。

### 2. 三路相机实时预览
- 来源：订阅 `config/cameras.yml` 中定义的 RealSense ROS2 话题（D435 顶视 + 2×D405 腕部）。
- 分辨率 640×480 @ 30 fps，叠加显示：实际帧率、端到端延迟、掉帧计数。
- 支持单路放大、画中画、暂停某一路预览（仅影响 UI 显示，不影响录制）。

### 3. 双臂状态可视化
- 订阅 `config/pipers.yml` 中 4 路 Piper 的关节反馈话题。
- 实时显示 14 维观测：左右臂各 7 个关节角 + 2 个夹爪开度。
- 组件：关节条形图、数值、电机温度/扭矩、CAN 链路心跳灯。
- 紧急停止按钮常驻顶栏（软件 disable + UI 锁定双重保护）。

### 4. 遥操作 / 录制控制（精简版）
- **只保留三种动作**：`开始`、`保存`、`丢弃`。
- 流程：选择任务 + Prompt → `开始` → 实时录制 → `保存`（成功写盘、编号自动递增）或 `丢弃`（清理临时文件、不递增）。
- **已移除**：暂停 / 继续、重录。录制中途若出错，统一走 `丢弃` → 重新 `开始`，简化状态机、避免半段数据。
- 录制源与 `scripts/start_teleop.sh` 复用同一套遥操作主从配置。

### 5. 任务与 Prompt（下拉选择，模板由管理员维护）
- 采集员界面为**只读下拉**：
  - 「任务」下拉：`Task_A/base`、`Task_A/dagger`、`Task_B/...` 等（由管理员启用项决定）。
  - 「Prompt」下拉：与所选任务关联的 prompt 列表，支持 AWBC 格式 `"<task>, Advantage: positive/negative"`。
- 选中后显示在"已选任务"只读区，并同步到状态栏。未选完整就 `开始` 会被拦截。
- 模板存储：后端 `config/collection_templates.yml`（新增），字段 `task_id / subset / prompt / enabled / note`，供前后端共享。
- 管理员入口：独立的「模板管理」面板（新增 / 编辑 / 启停用 / 排序），改动后采集员下拉实时刷新。
- 落地：prompt 写入 `meta/tasks.jsonl`，task / subset 决定存盘根路径。

### 6. Episode 元数据
- 操作员、场景/光照标签、成功/失败、自由备注。
- 落地到 `meta/episodes.jsonl`，配合 LeRobot v2.1 规范。

### 7. 存储与磁盘监控
- 目标路径预览：`kai0/data/Task_<X>/<base|dagger>/data/chunk-000/episode_*.parquet` 以及 `videos/chunk-000/<camera>/episode_*.mp4`。
- 实时显示剩余磁盘空间、写入速率、当前 episode 累计大小（同步到状态栏）。

### 8. 回放预览（当次 episode）
- 保存后在同一页回放三路视频 + 关节曲线，用于现场质检。
- 可一键标记该 episode 为「丢弃」（管理员模式下真正删除；采集员模式只打标，由管理员决定是否清理）。

### 9. 历史 Episode 浏览与回放（采集员可用）
- 侧栏列表展示 `kai0/data/Task_<X>/<base|dagger>/` 下所有 episode，字段：编号、时长、成功标记、操作员、prompt、采集时间、大小。
- 过滤与搜索：任务 / 子集 / 成功失败 / 操作员 / 日期范围 / prompt 关键字。
- 点击任意一条进入**回放面板**：
  - 同步播放三路 mp4 + 14 维 state/action 曲线（从 parquet 读取），时间轴可拖动。
  - 倍速（0.25×–4×）、逐帧前进/后退、A/B 区间循环。
  - 显示元信息（prompt、备注、质检指标）；采集员只读，管理员可编辑/删除。
  - 一键「以此任务 + Prompt 新建采集」：回填下拉选择，方便复采。
- 后端：读走与录制相同的 LeRobot loader，避免路径 / 格式漂移；视频走 HTTP Range 流（mp4 直出），关节曲线走 WS 或 HTTP 一次性加载。

### 10. 已录数据统计模块（独立模块，真实文件计数）
> **设计原则**：该模块**完全独立**，不信任任何录制流内存状态，只扫描磁盘上真实存在的 episode 文件，保证断电 / 崩溃 / 手工移动文件后计数依然准确。

- **数据源**：直接递归扫描 `kai0/data/Task_*/<base|dagger>/data/chunk-*/episode_*.parquet`，以该文件为「真实存在」的唯一凭据；再交叉核对 `videos/chunk-*/<camera>/episode_*.mp4` 三路视频是否齐全，缺失则记为「残缺」而非正常计数。
- **统计维度**：
  - 总条数 / 今日新增 / 本周新增。
  - 按 `Task / subset` 分组：如 `Task_A/base: 812`、`Task_A/dagger: 146`。
  - 按操作员分组、按 prompt 分组、按成功/失败分组（读 `meta/episodes.jsonl`）。
  - 总时长（秒 / 小时）、总大小（GB）。
  - 残缺 episode 计数（parquet 与三路视频不一致）。
- **实现**：
  - 后端独立服务 `stats_service.py`，暴露 `/api/stats`；不与录制 worker 共享状态。
  - 启动时做一次全量扫描 + 建索引（SQLite 或内存缓存）；随后用 `watchdog` 监听 `kai0/data/**` 的 `create/delete/move`，增量更新。
  - 提供手动「强制重扫」按钮，防止 watch 事件丢失。
  - 录制流保存完成后**也主动发一次增量触发**，作为双保险，但最终口径以磁盘扫描为准。
- **前端**：独立卡片（非录制面板子组件），展示总数 + 分组柱状图 + 今日曲线；采集员只读、管理员可跳转到「残缺 episode」清单处理。

### 11. 数据质检看板
- 自动统计：episode 时长、平均动作幅度、各相机掉帧、关节越界告警、双手同步偏差。
- 超阈值高亮提醒操作员复采。

## 三、数据与底层对接

| 项 | 方式 |
|---|---|
| 相机流 | 订阅 RealSense ROS2 话题（`config/cameras.yml`） |
| 机械臂状态/动作 | Piper ROS2 话题（`config/pipers.yml`） |
| 遥操作源 | 复用 `scripts/start_teleop.sh` 主从遥操作栈 |
| 任务 / Prompt 模板 | `config/collection_templates.yml`（新增，管理员维护） |
| 存盘格式 | LeRobot v2.1：14 维 state/action、`top_head/hand_left/hand_right` 三路视频、480×640 @ 30 fps |
| 统计数据源 | 扫描 `kai0/data/**` 真实 parquet / mp4 文件（独立 service） |
| 紧急停止 | CAN 侧 disable + UI 状态锁，双重保护 |

观测维度严格与 `src/openpi/policies/agilex_policy.py` 对齐，避免训练/部署分布错位。

## 四、技术栈建议

**推荐方案：React + FastAPI + rclpy 桥**

| 层 | 选型 | 备注 |
|---|---|---|
| 前端 | React + TypeScript + WebSocket | 图表用 `recharts`，视频流走 MJPEG 或 WebRTC |
| 后端 | FastAPI + `rclpy` | 订阅 ROS2 话题 → WS 推流；复用 `serving/websocket_policy_server.py` 风格 |
| 存盘 | `datasets` (LeRobot) / `pyarrow` + `av` | parquet 写 state/action，av 写 mp4 |
| 统计 | 独立 `stats_service.py` + SQLite 索引 + `watchdog` | 与录制 worker 完全解耦 |
| 可选替代 | Foxglove Studio + 自定义 panel | 开发量最小，但录制流程需自己补 |

## 五、开发顺序

1. **后端数据通路**：ROS2 ↔ WS 桥 + 录制服务（能正确落 parquet/mp4，通过 LeRobot loader 回读校验）。
2. **状态栏 + 只读看板**：三相机预览、双臂状态、顶部 StatusBar（健康/录制/磁盘）。
3. **录制闭环**：开始 / 保存 / 丢弃三按钮 + 元数据表单 + 磁盘监控。
4. **任务 / Prompt 模板系统**：`collection_templates.yml` + 管理员编辑面板 + 采集员下拉。
5. **统计模块（独立）**：磁盘扫描 + watchdog 增量 + `/api/stats` + 前端卡片。
6. **模式切换**：采集员 / 管理员双模式（无密码，UI 层权限守卫 + 后端 API 简单角色头）。
7. **历史 episode 浏览与回放**：列表 / 过滤 / 三路视频同步回放 / 曲线叠加 / 倍速 / 一键复采。
8. **Rerun 可视化（管理员专属）**：对接 `docs/deployment/inference_visualization*.md`，支持实时流与历史 episode 两种入口。
9. **质检看板**：阈值告警 + 复采提示。
10. **联调**：与 `scripts/start_teleop.sh` 串联，跑通一次完整 Task A 采集流程（采集员 + 管理员双角色 E2E）。

## 六、风险与注意事项

- **状态栏真值来源**：StatusBar 中的 Episode 编号、总数必须以独立统计模块的磁盘扫描结果为准，不能仅依赖录制 worker 计数，避免崩溃后错号。
- **时间戳对齐**：相机帧与关节反馈必须统一时间基（建议以 ROS2 时钟为准），否则 state/action 对齐偏差会污染数据。
- **30 fps 同步**：三路相机同步录制压力较大，后端建议异步写盘 + 环形缓冲。
- **Episode 原子性**：保存必须在所有流停止并 flush 完成后再写 `episodes.jsonl`，避免产生半截 episode；统计模块据此可以识别残缺条目。
- **与 χ₀ 训练输入一致**：14 维 obs 顺序、夹爪归一化、相机 key 名称须与 `agilex_policy.py` 完全一致。
- **模板变更的向后兼容**：管理员删除 / 重命名任务或 prompt 时，历史 episode 仍引用旧值，历史浏览需容忍"模板已删除"的情况，不得报错。
- **断网/断电恢复**：支持把最近一条未落盘 episode 标记为损坏并跳号；统计模块启动时会扫出并上报。
