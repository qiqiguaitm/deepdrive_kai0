# Training CLI Notes

Training-specific commands and environment notes extracted from the
pre-merge training workspace (gf0/gf1/gf2 side). Deployment/inference
commands live in `docs/deployment/` or the repo-root `CLAUDE.md`.

All commands below run from `kai0/` unless noted. Source `setup_env.sh`
at the repo root first to populate `KAI0_DATA_ROOT`, `OPENPI_DATA_HOME`,
and `PYTORCH_CKPT_BASE` for your machine profile (gf / sim01 / default).

```bash
source setup_env.sh
```

## Training

```bash
# JAX full fine-tuning, FSDP across 8 GPUs
uv run scripts/train.py <config_name> --exp_name=<name> --fsdp-devices 8 --batch-size 128

# PyTorch training (advantage estimator)
uv run python scripts/train_pytorch.py <config_name> --exp_name=<name> --save_interval 10000

# Multi-GPU DDP
uv run torchrun --standalone --nproc_per_node=8 scripts/train_pytorch.py \
    <config_name> --exp_name=<name>

# Compute norm_stats before first training on a dataset
uv run python scripts/compute_norm_states_fast.py --config-name <config_name>
```

## In-training eval (AWBC line — tensor-based)

Set these on the TrainConfig (see `pi05_flatten_fold_awbc`):

- `val_ratio=0.1` — 90/10 episode split
- `eval_interval_early=100`, `eval_interval_late=1000`
- `eval_early_count=3`
- `eval_batches=4`

Produces `eval/mae_joint_{1,10,50}` (rad) and `eval/mae_grip_{1,10,50}` (m)
in wandb.

## In-training eval (stand_box line — video-based)

Set these on the TrainConfig (see Task E configs):

- `inline_eval_val_root="/path/to/val"` — standalone val dataset root
  with `data/chunk-000/episode_*.parquet` + `videos/`
- `inline_eval_n_frames=20`, `inline_eval_every=1`

Produces `val/mae_{1,10,25,50}` in wandb. Reads mp4 frames sparsely;
~500 MB RAM per process vs ~18.5 GB for full-episode cache.

## AWBC prompt format

Training data (generated by `stage_advantage/annotation/discretize_advantage.py`)
and inference prompts must match. This repo uses **period separator**:

```
"Flatten and fold the cloth. Advantage: positive"
"Flatten and fold the cloth. Advantage: negative"
```

(NOT `, Advantage:` — the comma format is ONLY used at training-time
continuous advantage injection via `InsertAdvantageIntoPrompt` transform.)

## Model Arithmetic

```bash
# Dump validation data, then mix checkpoints
python model_arithmetic/dump_data.py --dataset <config> --output val.pkl
python model_arithmetic/arithmetic.py --config <config> --data-path val.pkl \
    --checkpoints /path/ckpt1 /path/ckpt2 --output /path/mixed \
    --optimize_method inverse_loss --use_gpu
```

Six merge methods supported: `average`, `inverse_loss`, `gradient_descent`,
`adaptive_gradient_descent`, `greedy`, `manual`.

## Multi-node training

Set these env vars before `uv run scripts/train.py`:

```bash
export JAX_COORDINATOR_ADDRESS=<host:port>
export JAX_NUM_PROCESSES=<total processes>
export JAX_PROCESS_INDEX=<this process's index>
```

See `train_scripts/launch/run_multinode.sh` for a full example.

```bash
KAI0_DATA_ROOT=/custom/path source setup_env.sh
```

Profile auto-detect: hostname match first, then filesystem probe
(`/vePFS/` → gf, `/data1/` → sim01), then default.
